# Filter to see single purchase transactions

[Filter to see single purchase transactions](/changelog/2023/filter-single-purchase-transactions#filter-to-see-single-purchase-transactions)

Get a list of transactions that aren't related to a subscription when working with the API. Use this to see transactions for single purchases.


## What's new?

[What's new?](/changelog/2023/filter-single-purchase-transactions#change-summary)

We updated the Paddle API so that now you can passnullas a value to thesubscription_idquery parameter whenlisting transactions. You can use this to filter to see single purchase transactions that aren't related to a subscription.

[listing transactions](/api-reference/transactions/list-transactions)

Return entities related to the specified subscription. Use a comma-separated list to specify multiple subscription IDs.


## How it works

[How it works](/changelog/2023/filter-single-purchase-transactions#change-details)

Transactionsandsubscriptionsare closely related entities in Paddle. For example:

[Transactions](/api-reference/transactions/overview)
[subscriptions](/api-reference/subscriptions/overview)
- Where a transaction for recurring items completes, Paddle automatically creates a related subscription.
- Subscriptions automatically create transactions for renewals,subscription lifecycle changes, andone-time charges.
[subscription lifecycle changes](/build/subscriptions/replace-products-prices-upgrade-downgrade)
[one-time charges](/build/subscriptions/bill-add-one-time-charge)

Where a transaction and a subscription are related, thesubscription_idis included against the transaction for your reference. You can filter to see a transactions for particular subscriptions using thesubscription_idquery parameter when listing transactions.


You can also use Paddle to sell single purchases that don't create new subscriptions or relate to an existing subscription. This might include ebooks, software downloads, or other digital goods. Where transactions are for single purchases, thesubscription_idisnull.


Previously, there was no way to filter to see single purchases in Paddle.


Now, you can passnullas a value when using thesubscription_idquery parameter whenlisting transactionsto see them. Paddle returns transactions that aren't related to a subscription. Combine with thestatusparameter with the valuecompletedto see transactions for single purchases.

[listing transactions](/api-reference/transactions/list-transactions)

Return entities that match the specified status. Use a comma-separated list to specify multiple status values.


Return entities related to the specified subscription. Use a comma-separated list to specify multiple subscription IDs.

> As well as single purchase transactions, transactions for incomplete checkouts and draft invoices havenullas a value forsubscription_id— even if the transaction is for recurring items. Paddle creates a subscription for recurring items when a checkout completes or when an invoice is issued.


As well as single purchase transactions, transactions for incomplete checkouts and draft invoices havenullas a value forsubscription_id— even if the transaction is for recurring items. Paddle creates a subscription for recurring items when a checkout completes or when an invoice is issued.
